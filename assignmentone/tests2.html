param test

This param tag is used to define parameters for plugins embedded with an object element. For example, below is an video file that has a param name autoplay and the value has been set to true. This will autoplay the video file when the page loads. Let's move to the groove!

video test

This video tag specifies video such as a movie clip or other video streams. There are 3 supported video formats for the video element. MP4, WebM, and Ogg. We have the same video used before except it doesn't have the autoplay parameter.

audio test

This audio tag defines sound, such as music or other audio streams. There are 3 supported file formats. MP3, Wav, and Ogg. Let's listen to some rad music now!

source test

This source tag is used to specify multiple media resources for media elements, such as video and audio. It allows you to specify alternative video/audio files which the browser may choose from, based on its media type or codec support. Below will either be audio or video depending on which your browser chooses/supports. This is useful because you don't know what browsers the users are using. It is helpful to provide multiple sources.

track test

This is a track tag that specifies the text that tracks for media elements. This element is used to specify subtitles, caption files, or other files containing text, that should be visible when the media is playing.

footer test

Footer tag defines a footer for a document or section. It should contain information about authorship, copyright, contact, sitemap, back to top links, related documents. It is typically located at the bottom of the document.
